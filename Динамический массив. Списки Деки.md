### Динамический массив

Асимптотика доступа к ячейке константная. Получение по индексу занимает O(1)
Менять размер массива нельзя

Нам хотелось бы иметь такие новые операции:
- удаление последнего числа
- добавление последнего числа

Что делает append()?
- Создается новый пустой массив
- Старый массив копируется в новый
- Новый элемент добавляется в конец

Что делает remove()?
- Создается новый пустой массив
- Старый массив копируется в новый (кроме последнего элемента)

Псевдокод:
Алгосложность {i}. O(1) времени
```
DynamicArray{
	data: []
	операция [i]:
		return data[i]
	
	append(e):
		new_data = [длина(data) + 1 нулей]
		скопируем data в new_data
		data = new_data
		data[конец] = e
		
	removeEnd():
		new_data = [длина(data) - 1 нулей]
		скопируем data без последнего элемента в new_data
		data = new_data	
}
```

Улучшенный подход добавления нового элемента:
- создаем массив в 2 раза больше (про запас)
- старый массив копируется в новый
- добавляем элемент в первую свободную ячейку из запаса

Псевдокод:
```
DynamicArray{
	data: [0]
	last: -1
	
	операция [i]:
		if <= last
			return data[i]
		else
			ошибка, выход за границы динамического массива
	
	append(e):
		if last + 1 = длина(data)
			new_data = [2*длина(data) нулей]
			скопируем data в new_data
		data = new_data
		data[last + 1] = e
		last += 1
}
```

Ключевое отличие: в среднем операция добавления стала O(1), хоть и в худшем случае O(n)
Такая асимптотика называется амортизационной, а процедура ее подсчета - амортизационным анализом


### Динамический массив. Задача

Рассмотрим задачу: Нужно создать пустой массив и последовательно добавить в него n чисел. Какова асимптотика такой программы?
Разберем наивный подход добавления элемента

Предположим, что мы уже добавили 4 элемента и хотим добавить 5-й элемент к старому массиву

1. Создается новый массив
2. Копируется в новый
3. Добавление в конец элемент
Вставка в конец наивного массива всегда занимает O(длина массива) времени

Также предположим, что мы уже добавили 4 элемента и хотим добавить 5-й элемент к старому массиву

1. Создаем новый массив в 2 раза больше
2. Копируем в новый
3. Новый элемент добавляется в свободную ячейку
Вставка в конец улучшенного динамического массива, не имеющего запаса, занимает O(длина массива) времени

Тогда в чем различие?

При добавлении 6-го элемента, используя наивный подход, всегда затрачивается O(длина массива) времени
При добавлении 6-го элемента, используя улучшенный подход, затрачивается O(1) времени


### Динамический массив. Удаление с конца

Улучшенный подход при удалении элемента
Рассмотрим массив, в котором после удаления запас станет половиной
- Создаем новый массив вдвое меньше
- Копируем из старого в новый

Почему такой подход считается плохим?
Потому что повторяя чередование append и removeEnd, мы будем попадать каждый раз на линейное расширение или уменьшение внутреннего масива, что превратит нашу амортища
