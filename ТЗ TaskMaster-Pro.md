Задание - [[Программирование]]

---
## 1. Цель

Разработать консольное TUI-приложение для управления задачами с хранением данных в CSV-файлах.

---
## 2. Общие требования

Сделать:
- Консольный TUI-интерфейс
- Хранение данных в CSV
- Изоляцию данных по пользователям (отдельный CSV на пользователя)
- Валидацию данных через Pydantic
- Цветовую схему через Colorama
- Использование только стандартной библиотеки + pydantic + colorama

---
## 3. Функциональность

Реализовать:
- Создание задачи
- Просмотр списка задач
- Просмотр деталей задачи
- Редактирование задачи
- Удаление задачи
- Фильтрацию по статусу
Поддержать:
- Приоритеты (4 уровня)
- Статусы (3 состояния)
- Навигацию по списку с клавиатуры
---
## 4. Хранение данных

Обеспечить:
- CSV-файл формата:  
    id, user_id, title, priority, status, created_at, updated_at, deadline
- Автоматическое создание файла
- Атомарную запись
- Thread-safe доступ
- Создание backup при изменении
## 5. Архитектурные требования

Обязательно использовать:
- Метакласс Singleton для TaskManager(user_id)
- Dataclass для модели Task
- ABC для слоя хранения
- Pydantic-модели для создания и обновления
- Дескрипторы
- Магические методы в TaskList
- Context manager
- F-строки
---
## 6. Валидация

Проверять:
- title — 1–200 символов, запрещённые спецсимволы
- priority — только 1–4
- status — только 0–2
- deadline — YYYY-MM-DD или пусто\

## 7. Оформление репозитория проекта

Сделать:
- Чёткую и логичную структуру каталогов согласно ТЗ
- Отдельную директорию `data/` для пользовательских CSV-файлов
- Наличие `README.md` с:
    - описанием проекта
    - списком возможностей
    - инструкцией по запуску
    - списком зависимостей
- Наличие `requirements.txt`
- Наличие `.gitignore` (исключить venv, **pycache**, временные файлы, CSV-данные при необходимости)
- Чистый и единообразный стиль кода
- Осмысленные коммиты
- Отсутствие лишних файлов и мусора в репозитории
- Рабочий проект после клонирования и установки зависимостей
