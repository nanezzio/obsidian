### Теория

**Алгоритм** - придуманная последовательность шагов для решения поставленной задачи

Пример: Написать программу, которая получает на вход массив и переставляет в нем элементы так, что он оказывается отсортированным
73319 -> 13379

Зачем изучать алгоритмику:

1. Мыслить как программист
2. Проектировать быстрые алгоритмы
3. Применять стандартные структуры данных
4. Успешно проходить собеседования


### Базовые понятия алгоритмики

- **Массив** - пронумерованным набор элементов фиксированного размера (нумерация с нуля)

При создании массива говорим что хотим 10 ячеек и в дальнейшем не можем поменять их размер

- **Псевдокод** - вымышленный условный язык программирования, предназначенный исключительно для демонстрации и чтения человеком, а не компьютером

Например:

Завести массив: arr = [1, 22, 3, 45, 7]
Условный оператор: 
if a > b
	print "a больше b"
else
	return "b больше a"

- **Алгоритмическая сложность** - сложность алгоритма для компьютера. Алгоритмы отличаются друг от друга потреблением памяти и времени


### Алгоритмическая сложность

У нас есть массив. Его длину обозначим n. Надо написать программу, которая ищет максимум

13 42 67 43 89 90 67 текущий максимум: 13
13 42 67 43 89 90 67 текущий максимум: 42
13 42 67 43 89 90 67 текущий максимум: 67
13 42 67 43 89 90 67 текущий максимум: 67
13 42 67 43 89 90 67 текущий максимум: 89
13 42 67 43 89 90 67 текущий максимум: 90
13 42 67 43 89 90 67 текущий максимум: 90

Прошли весь массив, ответ в текущем максимуме
Решение: 
```
arr = [1, 22, 3, 45, 7]  заводим или получаем на вход массив        
n = длина(arr)

max = arr[0]   первым кандидатом на максимум возьмем первый элемент массива

for i от 0 до n-1   пробежимся цилом по массиву

if arr[i] > max   если очередной элемент больше текущего максимума, то обновим
	max = arr[i]

print "Максимум:" max
```

**Инвариант** - утверждение, сохраняющее свою справедливость во время работы программы

Инвариант в алгоритме поиска максимума:
На каждой итерации в max максимальный среди всех просмотренных элементов


Как быстро работает программа?

Ответ в секундах не подойдет. На новом ноутбуке будет работать быстрее, чем на старом. Если я запущу параллельно слишком много программ, то, возможно, будет и наоборот. Так как же измерить время? В количестве простых операций? Тоже слишком муторно, для простых программ


Как правильно измерить время?

В алгоритмике измеряют, как сильно меняется время работы при увелечении размера входных данных

n                100n                    1000n
1 сек          100 сек                1000 сек

Время работы = O(n)

Такой характер роста (асимптотика) называется линейным


Память измеряется аналогично - фиксируется характер роста используемой памяти при увеличении размера входных данных

Доппамять                  Доппамять
n max/i/n                     100n max/i/n
		Константность = O(1)


Задача

У нас есть массив из замеров температур по одному замеру в день
Его длину (т.е. количество дней) обозначим n
Надо написать программу, находящую количество дней с неуникальной температурой

13 12 7 14 12 14 15 -> 4 дня

Решение:
```
arr = [1,22,3,45,7]   Заводим или получаем на вход массив данных
n = длина(arr)

count = 0  Заводим счетчик искомых дней

for i от 0 до n-1   Перебираем номера потенциальных неуникальных дней
	for j от 0 до n-1  Перебираем все, с кем можно совпасть
		if arr[i] == arr[j] И i != j
			увеличь count на 1
			завершаем внутренний цикл
			
print "Количество неуникальных дней:" count
```

**Двойной цикл** - квадратичное время

При увеличении данных в 10 раз у нас программа выполнит в 100 раз больше операций

n                              100n                            1000n
1 сек                       10000 сек                 1 млн сек

Такой характер роста (асимптотика) называется квадратичным

Память также константна


### Что такое двоичный логарифм

**Двоичный логарифм** - это то число, в которое надо возвести двойку, чтобы получить исходное число

С другой стороны, логарифм числа можно представить как **количество делений его пополам, чтобы получить 1**

Например, лог2 8:
- 8 : 2 = 4 - первый раз
- 4 : 2 = 2 - второй раз
- 2 : 2 = 1 - третий раз
Поэтому лог2 8 = 3


Пирамида с шириной 8 элементов будет иметь высоту 4
Это записывается как 1 + лог2 8 = 1 + 3 = 4

               2 2
             4 4 4 4 
            8 8 8 8 8 8 8
Количество элементов в пирамиде 2 * 8 - 1 =15


### Бинарный поиск

Задача

Для заданного числа X требуется найти номер ячейки с ним

1 3 4 6 7 8 10 13 14

Ответ: находится в ячейке номер 2

Решение:

Пройдемся по всем элементам массива

- Если мы наткнемся на искомый элемент, то сообщим пользователю, что он найден в рассматриваемой
- Если пройдем весь массив и не найдем его, то скажем, что элемента в массиве нет

```
for i от 0 до n-1
	if arr[i] = x
		нашли в i!
```

Алгосложность 

Доппамять все так же константная, время - O(n), так как каждый элемент массива мы рассматриваем по одному разу константной операцией (Сравнением на равенство с x)


Идея:
Если вас попросят найти слово в словаре, вы не станете постранично его читать с самого начала
Зная, что слова в словаре расположены в лексиографическом порядке, вы ткнете в середину словаря. Вам достаточно будет этой страницы, чтобы определить, слева или справа от нее надо искать слово
После чего, используя тот же подход, вы продолжите поиск в оставшейся части словаря

Псевдокод такого решения(заучить):

```
left = 0
right = длина(arr)-1

while left < right
	middle = (left+right)/2
	if arr[middle] = x
		нашли! выходим из цикла
	else if arr[middle] > x
		right = middle - 1 //ищем слева
	else if arr[middle] < x
		left = middle + 1 //ищем справа
if arr[left] = x
	нашли!
		
if нашли
	print "элемент есть"	
else
	print "элемента нет"
```

Алгосложность

Запускаем наш алгоритм на промежутке всего массива
На каждом шаге мы смотрим только на один серединный элемент после чего уменьшаем размер промежутка в два раза
Время работы O(logn), а не O(n), что гораздо быстрее


### Правила подсчета алгосложности

Повторы? Умножайте

Действие, выполняющееся за O(X) времени, повторяется Y раз, тогда, если мы поставим одно после другого, общая сложность O(X+Y)


Сложение нескольких? Вычеркните все, кроме быстрорастущего

Если сложность получается O(2^n * n +n^3 + n + logn), то надо записать просто O(2^n * n)

Как определить, кто быстрее растет? Примеры в порядке возрастания: 1, logn, n, n^2, n^3,...,
