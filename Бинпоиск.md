### Теория

**Алгоритм** - придуманная последовательность шагов для решения поставленной задачи

Пример: Написать программу, которая получает на вход массив и переставляет в нем элементы так, что он оказывается отсортированным
73319 -> 13379

Зачем изучать алгоритмику:

1. Мыслить как программист
2. Проектировать быстрые алгоритмы
3. Применять стандартные структуры данных
4. Успешно проходить собеседования


### Базовые понятия алгоритмики

- **Массив** - пронумерованным набор элементов фиксированного размера (нумерация с нуля)

При создании массива говорим что хотим 10 ячеек и в дальнейшем не можем поменять их размер

- **Псевдокод** - вымышленный условный язык программирования, предназначенный исключительно для демонстрации и чтения человеком, а не компьютером

Например:

Завести массив: arr = [1, 22, 3, 45, 7]
Условный оператор: 
if a > b
	print "a больше b"
else
	return "b больше a"

- **Алгоритмическая сложность** - сложность алгоритма для компьютера. Алгоритмы отличаются друг от друга потреблением памяти и времени


### Алгоритмическая сложность

У нас есть массив. Его длину обозначим n. Надо написать программу, которая ищет максимум

13 42 67 43 89 90 67 текущий максимум: 13
13 42 67 43 89 90 67 текущий максимум: 42
13 42 67 43 89 90 67 текущий максимум: 67
13 42 67 43 89 90 67 текущий максимум: 67
13 42 67 43 89 90 67 текущий максимум: 89
13 42 67 43 89 90 67 текущий максимум: 90
13 42 67 43 89 90 67 текущий максимум: 90

Прошли весь массив, ответ в текущем максимуме
Решение: 
```
arr = [1, 22, 3, 45, 7]  заводим или получаем на вход массив        
n = длина(arr)

max = arr[0]   первым кандидатом на максимум возьмем первый элемент массива

for i от 0 до n-1   пробежимся цилом по массиву

if arr[i] > max   если очередной элемент больше текущего максимума, то обновим
	max = arr[i]

print "Максимум:" max
```

**Инвариант** - утверждение, сохраняющее свою справедливость во время работы программы

Инвариант в алгоритме поиска максимума:
На каждой итерации в max максимальный среди всех просмотренных элементов


Как быстро работает программа?

Ответ в секундах не подойдет. На новом ноутбуке будет работать быстрее, чем на старом. Если я запущу параллельно слишком много программ, то, возможно, будет и наоборот. Так как же измерить время? В количестве простых операций? Тоже слишком муторно, для простых программ


Как правильно измерить время?

В алгоритмике измеряют, как сильно меняется время работы при увелечении размера входных данных

n                100n                    1000n
1 сек          100 сек                1000 сек

Время работы = O(n)

Такой характер роста (асимптотика) называется линейным


Память измеряется аналогично - фиксируется характер роста используемой памяти при увеличении размера входных данных

Доппамять                  Доппамять
n max/i/n                     100n max/i/n
		Константность = O(1)
