### Рекурсия

Рекурсия - вызов внутри одной функции себя самой, чаще всего с другими параметрами

Пример:
Рекурсивная функция подсчета перемножения двух положительных целых чисел
- Решение одного случая легко выразить через решение для другого случая 
-  Рекурсивные вызовы когда-то точно закончатся

```
mult(a,b):
	if b == 1
		return a
	else
		return a + mult(a,b-1)
print "Ответ:" mult(n)
```

Не сработает:
```
mult(a,b):
	if b == 1
		return a
	else
		return mult(a,b-1)+a
print "Ответ:" mult(n)
```


### Задача о монетах

Дано: число - денежная сумма, которую надо собрать из монет номиналом 3 или 5 рублей

Надо: определить, можно ли собрать эту сумму

Пример:

7 - нельзя
17 - можно (3, 3, 3, 3, 5)

Эта задача легко решается с помощью рекурсивного алгоритма. Мы можем убедиться, что алгоритм корректный и рекурсия остановится
```
can(sum):
	if sum = 3 или 5 
		return ДА
	if sum >= 3 и can(sum - 3)
		return ДА
	if sum >= 5 и can(sun - 5)
		return ДА
	return НЕТ
```

Заметьте: с глубиной количество рекурсивных вызовов быстро растет, но сами вызовы начинают повторяться. Глубина от n/5 до n/3

Алгосложность:

Глубина в лучшем случае n/5. У каждого рекурсивного вызова (кроме самых глубоких) есть по два своих рекурсивных вызова
В итоге имеем O(2^n/5) времени работы, то есть при увеличении суммы всего на 5 рублей время работы увеличится в 2 раза

Рассмотрим решение задачи методом рекурсии с запоминанием: это применение динамического программирования к рекурсии
```
memory = [sum нулей]

can(sum):
	if что-то лежит в memory[sum]:
		return memory[sum]
	if sum >= 3 И can(sum - 3)
		memory[sum] = ДА
		return ДА
	if sum >= 5 и can(sum - 5)
		memory[sum] = ДА
		return ДА
	memory[sum] = НЕТ
	return НЕТ
```


### Range minimum requests

Задача: у нас есть массив. Нужно написать программу, которая будет принимать запросы в виде двух индексов и искать минимальное значение на промежутке в массиве от первого индекса включительно до второго не включительно

Массив: 
0 4 2 5 66 4 25
минимум на промежутке {0,4,2} - 0
Запрос: 0 3 --> 0
{0,4,2,5,66,3,25}, ответ 0

```
arr = ...

min(left, right):
	ans = arr[left]
	for i от left до right
		if arr[i] < ans
			ans = arr[i]
	return ans
```

Характеристики этого решения, которые мы хотим улучшить:
- Время O(N)
- Доппамять O(1)

Исправленная версия:

```
arr = ...

memory = [n x n нулей]

for left от 0 до n
	min = arr[left]
	for right от left+1 до n
		if arr[right] < min
			min = arr[right]
		memory[left][right] = min
		
min(left, right):
	return memory[left][right]
```

Заметим, что у нас теперь две фазы работы алгоритма. Помимо фазы ответов на запросы, у нас есть подготовительная фаза, в которой мы заранее считаем все суммы. Такая фаза называется препроцессингом (предподсчетом)

Алгосложность:

Препроцессинг тратит времени O(n^2) и памяти O(n^2) из-за двумерного массива, в котором запоминает результат

Хватит ли памяти?

Работа фазы запросов ускорилась. Для массива длиной в 1 миллиард алгоритм займет 930 тысяч терабайт памяти, а квадратичное время предподсчета - больше трех лет. Это очень много

**SQRT - композиция**

arr = ...
memory = [n^0.5 нулей]

for left от 0 до n с шагом n^0.5
	min = arr[left]
	for right от left+1 до left + n^0.5
		if arr[right] < min

	memory[номер шага] = min
