### Теория 

Магические методы (они же dunder-методы) - это специальные методы классов Python, имена которых начинаются и заканчиваются двойным подчеркиванием: `__init__`, `__str__`, `__len__`, `__add__` и т.д. Они не вызываются напрямую в обычном коде, а автоматически срабатывают при использовании встроенных операций: `+`, == , `len()`, `str()`, `for` и т.д. 

### Зачем они нужны?

Идея: они позволяют объектам вести себя как встроенные типы. Вместо того чтобы делать свои функции типа `vector_add(a, b)` и `playlist_to_string(p)`, можно писать `a + b` и `str(p)`, а внутри будет вызываться `__add__` и `__str__` написанного класса. Это и есть перегрузка операторов: определяется, как объект будет реагировать на языковые операции, вместо придумывания отдельного API

**Важно**: магические методы - не "магия" в смысле черного ящика, а просто стандартный протокол: для каждой операции интерпретатор знает, какой метод попробовать вызвать у объекта

### Общий принцип работы

Когда Python встречает какую-то операцию, он делает примерно так:
 - `obj()` - вызывает `obj.__call__()`
 - `str(obj` - вызывает `obj.__str__()` (если нет - `__repr__()`)
 - `len(obj)` - `obj.__len__()`
 - `obj1 + obj2` - `obj1.__add__(obj2)`
 - `obj[i]` - `obj.__getitem__(i)`
 - `for x in obj` - сначала `obj.__iter__()`, затем у итератора `__next__()`

То есть обычный синтаксис - это просто "сахар" над вызовом этих специальных методов


### Создание и инициализация объекта: `__new__`, `__init__`

 - `__new__(cls, ...)` - инициализирует уже созданный объект (обычно не трогаешь, если не лезешь в метапрограммирование)
 - `__init__(self, ...)` - инициализирует уже созданный объект; здесь ты задаешь атрибуты экземпляра

Пример:

```python
class Point:
	def __init__(self, x, y):
		self.x = x
		self.y = y
		
p = Point(2, 5) 
# Python делает: obj = Point.__new__(Point); obj.__init__(2, 5)
```


### Деструктор

В Python "деструктор" - это магический метод `__del__(self)`, который вызывается, когда объект собираются уничтожить (точнее, когда интерпретатор решает, что объект больше не нужен и его можно финализировать) 

В классическом ООП деструктор - "обратная сторона" конструктора: конструктор подготавливает объект к жизни, деструктор делает финальную уборку перед смертью объекта. В Python роль деструктора выполняет метод `__del__(self)`, который можно определить в своем классе; он будет автоматически вызван, когда объект попадает под уничтожение (обычно при обнулении счетчика ссылок или при завершении программы)

Сигнатура минимальна:

```python
class Resourse:
	def __init__(self, name):
		self.name = name
		print(f"Ресурс {name} создан")
		
	def __del__(self):
		print(f"Ресурс {self.name} освобожден")
```

Здесь `__del__` - деструктор: когда экземпляр `Resource` будет уничтожен, выведется сообщение

Когда вызывается `__del__`
Python использует подсчет ссылок плюс сборщик мусора

### Основные сценарии

1. Когда счетчик ссылок объекта становится 0 (на него больше никто не ссылается)
2.  Когда программа завершает работу: оставшиеся объекты финализируются
3.  Когда явно удаляется последняя ссылка через `del` при условии, что ссылок на объект больше нет

```python
class Example:
    def __init__(self, name):
        self.name = name
        print(f"Создан объект {name}")

    def __del__(self):
        print(f"Уничтожен объект {self.name}")

obj1 = Example("test")
obj2 = obj1   # ещё одна ссылка
del obj1      # объект не уничтожен, есть obj2
del obj2      # теперь ссылок нет, вызовется __del__
```

Здесь деструктор реально срабатывает только после удаления **последней** ссылки

Важно: оператор `del obj` удаляет ссылку, а не гарантированно "прямо сейчас убивает объект". Если есть другие ссылки, объект живет дальше

### Для чего обычно используют деструктор

В теории - для освобождения ресурсов: закрыть файл, разорвать соединение с базой, освободить сокет, удалить временный файл и т.п.
